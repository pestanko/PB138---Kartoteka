Vytvorenie nového UI

Figure 1. Nové UI

Po otestovaní funkcionality backendu s pôvodným užívateľským prostredím vznikla potreba prehladnejšie a užívateľsky prívetivejšie zobrazovať dáta z kartotéky. Z tohoto dôvodu som sa pri návrhu nového UI rozhodol čo najviac priblížiť dobre známemu vzhľadu, známeho najmä z rôznych kancelárskych balíkov. UI bolo dodatočne ešte v neskoršej fázi vývoja upravené tak, aby sa správalo responzívne.

Mojou úlohou bolo naimplementovať dialógy a akcie spojené s úpravou dokumentu pomocou UI.

Figure 3. FileChooser Ubuntu

Dialógy pre zvyšné akcie som navrhol osve tak, aby každý dialóg vyhovoval požadovanej funkcionalite a neobsahoval/nevykonával nič viac, ako je potreba.

Validácia vstupov a kontrola akcií

Na princípoch defenzívneho programovania som overil platnosť požadovaných vstupov v dialógových oknách.

Takisto som vhodnými podmienkami zabránil spusteniu nežiadaných akcií od užívateľa (napr. vytvorenie kategórie, pokiaľ užívateľ nemá otvorený žiaden zošit).
